================================================================================
  OPTION 2: FULL INSTALLATION (For Real Network Monitoring)
================================================================================

After you've seen the test version and want to use it for real:

STEP 1: Install Npcap (for network scanning)
------
1. Download from: https://npcap.com/#download
2. Run installer as Administrator
3. CHECK the box: "Install Npcap in WinPcap API-compatible Mode"
4. Finish installation
5. RESTART YOUR COMPUTER

STEP 2: Install Dependencies
------
After restart, run:
   pip install flask flask-cors python-dotenv
   pip install scapy
   pip install pyodbc

If pyodbc fails:
- Download pre-built wheel from: https://www.lfd.uci.edu/~gohlke/pythonlibs/#pyodbc
- Or run: install_step_by_step.bat

STEP 3: Setup Database
------
1. Copy config/env.example to .env
2. Edit .env with your SQL Server details
3. Run config/database_setup.sql in SQL Server

STEP 4: Start Full Version
------
Right-click scripts\start_web_dashboard.bat ‚Üí Run as administrator

================================================================================
  TROUBLESHOOTING
================================================================================

Q: "pip is not recognized"
A: Use: python -m pip install flask flask-cors

Q: Flask installs but test won't start
A: Make sure you're in the project folder, then: python test_web_only.py

Q: Browser shows "can't connect"
A: Make sure the test server is running (you should see messages in terminal)

Q: pyodbc won't install
A: That's OK! Use the test version first. Fix pyodbc later when you want the full version.
   See: INSTALLATION_GUIDE_PYTHON_314.md for solutions

Q: Want to use it on my phone?
A: 1. Start the test server
   2. Find your PC's IP (run: ipconfig)
   3. On phone: http://YOUR-PC-IP:5000
